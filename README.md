# 임용 국어 4레이어 시스템 (검증판)

한 과목(국어)으로 4개 레이어 전체를 세로로 완성한 버전.
이게 돌면 → 같은 틀을 다른 과목(수학·사회…)으로 복제해 가로 확장한다.

## 설계 핵심 (합의된 원칙)
- **임베딩은 한 번만 학습해 고정**(`embeddings/gukeo_emb.pkl`). 이전에 학습이
  '퍼진' 원인(임베딩 매번 재생성)을 차단. 모든 레이어가 같은 좌표계 공유.
- **과목별로 SOM 분리 학습**. 전과목 한 지도에 부으면 발산 → 국어만 담는다.
- **출처 태그 강제**. `source` 없는 데이터는 스키마에서 거부됨.
- **레이어4는 예측기가 아니라 연습문제 생성기.** 해설은 실제 자료 출처를
  밝혀서만 나오고, `reveal()`(버튼) 전엔 잠겨 있다(먼저 풀게 강제 = 인출연습).

## 폴더
```
core/         schema.py(출처강제) · embedding.py(고정임베딩) · som.py · korean_tokenizer.py
layer2_corpus/ layer2.py  ← 자료 매핑 + 정합성 가드레일 (여기부터 학습)
layer1_pattern/layer1.py  ← 기출 시계열 패턴 + 급별(초/중/특) 교차분석
layer3_trend/  layer3.py  ← 검색어 생성 + Brave + 가드레일 필터 (키는 로컬에서)
layer4_generate/layer4.py ← 문제 생성 + 출처 해설 + reveal 잠금
data/         gukeo_L2.json(국어자료) · gukeo_L1_exams.json(기출)
embeddings/   학습 산출물(gukeo_emb.pkl, gukeo_som.pkl)
```

## 실행 순서
```bash
# 0) (선택) 샘플로 먼저 파이프라인 확인
python make_sample_data.py

# 1) 레이어2: 임베딩 고정 학습 + 국어 SOM 학습  ← 반드시 먼저
cd layer2_corpus && python layer2.py ../data/gukeo_L2.json

# 2) 레이어1: 기출을 그 SOM에 얹어 연도·급별 패턴 분석
cd ../layer1_pattern && python layer1.py ../data/gukeo_L1_exams.json

# 3) 레이어3: 검색어 생성(→ 로컬에서 Brave 키 꽂아 검색·필터)
cd ../layer3_trend && python layer3.py

# 4) 레이어4: 개념영역별 연습문제(해설 잠금). OpenAI 키 꽂으면 실제 생성
cd ../layer4_generate && python layer4.py
```

## 데이터 업로드 규칙 (중요)
모든 자료는 아래 태그를 붙여 `Record`로 만든다. 분류 체계를 먼저 지키고 올릴 것.
- `layer`   : L1_pattern / L2_corpus / L3_trend / L4_generate
- `subject` : 국어 (지금은 국어만; 복제 시 여기만 바꿈)
- `source`  : **필수** — 파일명/책+페이지/URL 등 출처
- `year`    : 기출이면 필수(L1)
- `level`   : 초등/중등/특수/공통 (기출 교차분석용)
- `code`    : 성취기준 코드(있으면 매핑 정확도↑)

## 다른 과목으로 복제할 때
`subject`만 바꾸고(예: "수학"), 그 과목 자료로 1~2단계를 다시 돌리면
`수학_emb.pkl` / `수학_som.pkl`이 생긴다. 코드는 그대로.

## 실데이터 팁
- 임베딩 어휘가 적으면(작은 코퍼스) `min_count`를 1~2로. 자료가 커지면 2 이상.
- SOM `grid`는 자료량에 맞춰: 수백 건이면 (10~14)², 수천 건이면 더 크게.
- 레이어1 패턴은 기출이 개념영역당 2건 이상 쌓여야 보인다 → 기출을 많이 넣을수록 좋음.

## 학습이 '퍼지지' 않게 하는 장치 (재발 방지)
1. 임베딩은 최초 1회만 학습해 고정 → 이후 모든 레이어가 같은 좌표계 공유.
2. 과목별 SOM 분리 → 전과목 혼합으로 인한 발산 차단.
3. 자료를 크게 바꿀 때만 'force_emb' 체크로 임베딩 재학습.

## 파일로 자료 넣기 (pdf / docx / txt)
각 탭의 "📄 파일로 넣기"에서 업로드:
- **추출**: pdf(pdfplumber) · docx(문단 + **표 안 성취기준**까지) · txt
- **자동 분할**: 성취기준 코드([4국05-04]) 앞에서 항목 분리, 종결어미 기준 문장 분할,
  페이지번호 등 노이즈 제거
- **코드 자동 감지**: 항목에 성취기준 코드가 있으면 자동으로 코드 칸에 채움
- **검토 후 저장**: 추출 결과를 편집표로 보여줌 → 제목·머리말 등은 체크 해제,
  문장 수정 가능 → '체크한 항목 저장'으로 pkl에 반영
- **출처는 파일 단위로 한 번 입력**(필수). L1 기출은 연도·급도 파일 단위로 지정.

> 자동 분할은 완벽하지 않다. 특히 표·다단 레이아웃 PDF는 추출이 흐트러질 수 있으니
> 편집표에서 손보는 단계를 반드시 거칠 것.

## 나와 함께 크는 학습 루프 (L4 확장)
문제를 풀면서 약점을 쌓고, 검토로 시스템을 개선한다. 데이터는 `{과목}_study.pkl`.

### 루프1 — 내 답 채점 → 약점 지도
1. 문제 풀고 **✍️ 내 답** 입력
2. **🤖 채점 제안**: LLM(키 있으면 모범답안 대조) 또는 정합도로 맞음/애매/틀림 *제안*
3. **내 확정(맞음/틀림)**: ★이것만 약점지도에 반영★ (자동 제안은 참고용)
   - 약점은 **성취기준 코드별 + 개념영역별** 정답률로 누적
4. **약점 위주 출제** 체크 시 → 정답률 낮은 영역에서 문제를 더 냄(적응형)

> 왜 '내 확정'만 반영하나: 자동 채점은 개념어 겹침/LLM 판단이라 서술형 논리를
> 100% 못 가린다. 부정확한 자동값이 약점지도를 오염시키지 않도록, 최종 판정은
> 사람이 쥔다(= GasCore의 인간 보증자 원칙).

### 루프2 — 문제·해설 검토 → 자료 개선 (복구 가능)
- 해설 연 뒤 **👍 좋은 문제 / 👎 이상한 문제·해설**로 그 개념영역 자료 신뢰도 조정
- 신뢰도 낮은 자료는 다음 문제 생성에서 후순위
- **↩️ 방금 검토 복구**: 잘못 누른 페널티 되돌리기 (좋은 자료가 묻히지 않게)

### 저장 파일
- `{과목}_study.pkl` : 약점지도(node/code별 정답률) + 채점이력 + 자료신뢰도 + 검토이력

## 기출 입력: 초등 vs 중등 (구조가 다름)
- **초등**: 한 시험에 전과목이 섞여 나옴 → 과목으로 못 나눔.
  → **연도 단위로 통짜 시험지(PDF)를 업로드** → 시스템이 문항 경계로 분할.
  - 문항 분할: 규칙 기반(문항번호 1./문 3/[3~4] 등) 기본, LLM 분할 옵션(더 정확).
  - 분할 결과를 편집표에서 검토(제목·안내문 제외, 묶음문항 행 나눔).
  - ⚠️ 스캔본 PDF(텍스트 없음)는 OCR 필요 — 앱이 감지해 알려줌.
  - (초등 통짜 기출의 과목 자동분류는 다른 과목 SOM이 갖춰지면 추가 가능)
- **중등/특수**: 과목별 시험 → 과목 선택 후 그대로 업로드.
- 급별(초/중/특) 교차분석은 문항의 과목이 정해진 뒤 가능.

## 전체 구조 (국어 완성판 · 6탭)
사이드바에서 과목 선택 → 6개 탭:

### 📚 자료·학습 (L2) — 자료종류별 태깅
자료 4종류를 구분해 넣는다(성격이 다름):
- **교육과정_성취기준**: 코드 자동 감지
- **지도서_각론**: 개념이 핵심. 학년군·영역·단원·학습모형 + **개념 태그**를 직접 붙임
  (개념 태깅 = 공부. 개념 사전에서 골라 표기 일관성 유지)
- **지도서_총론**: 그 과목 교수법·평가
- **교육과정_총론**: 전 교과 공통 → `공통_교육과정총론.pkl`에 별도 저장
- 이후 임베딩 고정 + 과목 SOM 학습(발산 방지)

### 📈 기출 패턴 (L1) — 속기사
기출 시계열 패턴 분석 + **🖋️ 경향 해설(속기사)**:
시스템(SOM)이 계산한 출제 수치를 LLM이 '문장으로만' 옮김(판단 아님, 번역).

### 🔎 트렌드 (L3) — 검색 + 사람 검토
검색어 자동 생성 → Brave 검색 → 속기 요약 → **반영/제거 검토** → 반영분만 저장.

### 📝 문제 풀기 (L4)
문제만 생성(해설은 reveal 시점 생성) → 내 답 채점 제안 → **내가 확정**(약점지도) →
해설 보기 → 문제·해설 검토(자료 신뢰도, 복구 가능).

### 🎯 수능형 연습 (L5) — 순수 연습
출처 없이 제시문 해석해 푸는 문제. 개념 지도와 독립. **유형별 약점**만 쌓음.

### 🕸️ 개념 지도
각론 개념(원천) → 지도서 총론·교육과정에서 같은 개념이 어디에 응용되나 표시.
= 출제 구조(각론 개념을 총론·교육과정에서 응용) 시각화.
